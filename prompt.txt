⚙️ Contexte du projet :
Application Dashboard déjà existante, construite avec gestion des utilisateurs, journal des changements, page automatisation, page paramètres Raspberry Pi (avec carte relais et sondes de température).
Base de données déjà en place (modèles utilisateur, historique, etc.).
Framework CSS : TailwindCSS
Backend : Python (Flask ou FastAPI, à préciser selon le projet existant)
Communication avec le Raspberry Pi en local via RPi.GPIO, w1thermsensor et adafruit-circuitpython-am2315.
🎯 Objectif de la nouvelle fonctionnalité
Créer la partie collecte, automatisation et visualisation dans le Dashboard :
Collecte automatique des données des sondes :
Température intérieure : capteur DS18B20 (bus 1-Wire)
Température et humidité extérieure : capteur AM2315 (I²C)
Contrôle des relais via des règles d’automatisation déjà définies dans la page Automatisation.
Stockage en base de données des mesures (température, humidité, état des relais).
Affichage sur le Dashboard :
Graphique des températures et humidité (jour, semaine, mois)
Indicateur d’état en temps réel (valeurs actuelles + statut des relais)
Bouton manuel ON/OFF pour chaque relais (avec mise à jour en base et commande GPIO)
Historique court (ex. dernières 10 mesures)
🧩 Détails matériels
🔌 Carte relais (contrôlée par RPi.GPIO)
Relay_Ch1 = 26
Relay_Ch2 = 20
Relay_Ch3 = 21
🌡️ Capteurs
Température intérieure (DS18B20)
Bus : w1
Librairie : w1thermsensor
Accès typique : from w1thermsensor import W1ThermSensor
Température / Humidité extérieure (AM2315)
Bus : I²C (SCL/SDA)
Librairie : adafruit-circuitpython-am2315
Accès typique :
import board, busio
import adafruit_am2315
i2c = busio.I2C(board.SCL, board.SDA)
sensor = adafruit_am2315.AM2315(i2c)
🧮 Schéma logique côté serveur (Python)
1. Services de collecte (cron ou thread)
Un script (ou service d’arrière-plan) lit régulièrement les sondes :
DS18B20 : sensor.get_temperature()
AM2315 : sensor.temperature, sensor.relative_humidity
Enregistre les données dans la base :
Table sensors_data (
  id INT PRIMARY KEY,
  timestamp DATETIME,
  indoor_temp FLOAT,
  outdoor_temp FLOAT,
  outdoor_humidity FLOAT,
  relay1_state BOOLEAN,
  relay2_state BOOLEAN,
  relay3_state BOOLEAN
)
2. Gestion des relais
Fonctions Python :
import RPi.GPIO as GPIO
GPIO.setmode(GPIO.BCM)
for pin in [26, 20, 21]:
    GPIO.setup(pin, GPIO.OUT)

def set_relay(channel, state):
    GPIO.output(channel, GPIO.HIGH if state else GPIO.LOW)
Ces fonctions sont appelées soit :
Par la page Automatisation (règles),
Soit par la page Dashboard (contrôle manuel).
